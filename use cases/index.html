# Сценарій: Створення нового облікового запису (UserSignUp)

ID: UserSignUp
Назва: Створення нового облікового запису
Учасники: Користувач, Система
Передумови: Користувач ще не має облікового запису
Результат: Користувач успішно зареєстрований
Виключні ситуації: - Введені некоректні дані → InvalidInputException
- Не заповнені обов’язкові поля → MissingDataException
- Вказана пошта вже використовується → DuplicateAccountException
Основний сценарій: 1. Користувач відкриває форму реєстрації.
2. Вносить особисті дані (ім’я, email, пароль тощо).
3. Система перевіряє коректність введених даних.
4. Система перевіряє, чи така пошта вже зареєстрована.
5. Якщо все гаразд, система створює обліковий запис.
6. Користувач отримує підтвердження про успішну реєстрацію.

# Сценарій: Вхід у систему (UserSignIn)

ID: UserSignIn
Назва: Вхід у систему
Учасники: Користувач, Система
Передумови: Користувач має активний акаунт
Результат: Користувач успішно авторизований
Виключні ситуації: - Невірний пароль → IncorrectPasswordException
- Обліковий запис не знайдено → AccountNotFoundException
- Перевищено кількість невдалих спроб → TooManyLoginAttemptsException
Основний сценарій: 1. Користувач відкриває сторінку входу.
2. Вводить email і пароль.
3. Система перевіряє відповідність даних.
4. Якщо вони правильні, користувач отримує доступ.
5. Система повідомляє про успішний вхід.

# Сценарій: Зміна рівня доступу користувача (UserUpgradeRole)

ID: UserUpgradeRole
Назва: Зміна рівня доступу користувача
Учасники: Адміністратор, Система
Передумови: Адміністратор має право змінювати ролі
Результат: Користувач отримує новий рівень доступу
Виключні ситуації: - Недостатньо прав у адміністратора → AccessDeniedException
- Користувач не існує → UserNotFoundException
- Користувач вже має найвищу можливу роль → RoleLimitReachedException
Основний сценарій: 1. Адміністратор вибирає користувача для зміни ролі.
2. Обирає новий рівень доступу.
3. Система перевіряє можливість зміни.
4. Якщо все добре, новий статус збережено.
5. Користувач отримує повідомлення про оновлення ролі.

# Сценарій: Оновлення профілю користувача (UserProfileUpdate)

ID: UserProfileUpdate
Назва: Оновлення профілю користувача
Учасники: Користувач, Система
Передумови: Користувач авторизований у системі
Результат: Змінені особисті дані профілю
Виключні ситуації: - Некоректні введені дані → InvalidProfileDataException
- Спроба змінити дані іншого користувача → PermissionDeniedException
Основний сценарій: 1. Користувач переходить у розділ “Профіль”.
2. Вносить зміни (ім’я, прізвище, фото, мову інтерфейсу тощо).
3. Система перевіряє дані.
4. Система зберігає оновлення.
5. Система сповіщає про успішне оновлення.

# Сценарій: Відновлення паролю (UserPasswordReset)

ID: UserPasswordReset
Назва: Відновлення паролю
Учасники: Користувач, Система
Передумови: Користувач зареєстрований у системі
Результат: Новий пароль встановлено
Виключні ситуації: - Email не зареєстровано → EmailNotFoundException
- Токен скидання недійсний → TokenInvalidException
Основний сценарій: 1. Користувач відкриває форму відновлення паролю.
2. Вводить свою email-адресу.
3. Система надсилає лист з токеном.
4. Користувач відкриває посилання з листа.
5. Вводить новий пароль.
6. Система змінює пароль та повідомляє про успіх.

# Сценарій: Деактивація користувача (UserRemove)

ID: UserRemove
Назва: Деактивація користувача
Учасники: Адміністратор, Користувач, Система
Передумови: Користувач має акаунт у системі
Результат: Обліковий запис заблоковано або видалено
Виключні ситуації: - Недостатньо прав — AccessDeniedException
- Користувач не знайдений — UserNotFoundException
Основний сценарій: 1. Адміністратор відкриває список користувачів.
2. Обирає того, кого потрібно видалити.
3. Система перевіряє права доступу.
4. Система деактивує акаунт або видаляє його.
5. Адміністратор отримує підтвердження про успішне видалення.

# Сценарій: Додавання нового матеріалу (ContentCreate)

Назва: Додавання нового матеріалу

2. Заповнює форму: заголовок, опис, файл.
3. Система перевіряє відповідність вимогам.
4. Якщо все добре, матеріал збережено.
5. Користувач отримує повідомлення про успішне створення.
